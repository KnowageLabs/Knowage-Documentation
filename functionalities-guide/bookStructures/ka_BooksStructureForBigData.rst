1.1 Documentation structure
---------------------------

KnowAge documentation is composed by:

-  **Installation Manual**;

-  **General User Manual**;

-  **General Administrator Manual**;

-  one more book for each product purchased.

The books focusing on products describe all the BI functionalities
included in your installation. For example if you have purchased
KnowageBD subscription, you will get the installation and the two
general manuals plus a third one which focus on the BI functionalities
provided by this product.

Target
------

Each book is addressed to different users.

-  **Installation Manual**: it is conceived for technical users. It
      provides all the needed instructions to install and configure the
      Knowage suite. Here the essential steps to succeed with the
      standard installation on certified environments will be described,
      including the optional use of CAS as a SSO solution and the use of
      HTTPS protocol.

Documentation structure

-  **General User Manual**: it is conceived for end users. It provides a
      first approach to Knowage interface and functionalities. It
      describes all the functionalities shared between each module that
      involves the end user.

-  **General Administrator Manual**: it is conceived for Knowage
      admnistrator. It describes all the management and configuration
      tools shared by all Knowage products.

-  **Product Manuals**: as we mentioned above, each one of these manuals
      is focused on a specific module/sub-product. They are conceived
      for both at end users and Knowage developers. Almost each chapter
      is divided in two parts: the first is useful to both end users and
      expert users, while the second one is for the expert users only.
      If you’d like to get a general but complete introduction to all BI
      functionalities you can read all the parts not marked with \*.
